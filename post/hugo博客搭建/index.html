介绍 hugo 博客简单的搭建过程，基础命令以及如何部署到 GitHub 远端

安装#

二进制（跨平台）#

从Hugo Releases下载适合平台的最新版本，这里以0.100.0版本为例，如 64 位的 Linux 可以下载hugo_0.100.0_Linux-32bit.tar.gz，Windows 和 MacOS 类似

下载好后，解压缩并移动到/usr/local/bin目录下

tar -zxvf hugo_0.100.0_Linux-32bit.tar.gz
mv hugo /usr/local/bin/

macOS#

macOS 可以使用 Homebrew直接安装（前提是安装了Homebrew）

brew install hugo

其它#

使用hugo version查看 Hugo 版本，检查是否成功安装

更多安装方法和详细过程建议阅读官方文档：Install Hugo

附上 GitHub 地址：hugo

部署#

新建站点#

找一个合适的目录（例如家目录~/），新建站点，siteName为站点目录名称

hugo new site siteName

默认站点使用toml格式的配置文件，加上-f参数，配置文件使用 yml 格式

hugo new site siteName -f yml

添加主题#

不同主题安装方式不一定相同，可以去搜索相应主题的文档

这里以PaperMod为例

在站点目录（刚才新建站点生成的文件夹）下

git clone https://github.com/adityatelange/hugo-PaperMod themes/PaperMod --depth=1

官方文档有更多方法及主题更多介绍：hugo-PaperMod

下载完成后，打开配置文件config.yml，添加主题配置项

theme: PaperMod

主题更新方式：

cd theme/PaperMod
git pull

添加文章#

在站点目录下，新建一篇文章，如first-post.md

hugo new post/first-post.md

文件会生成到content/post/first-post.md位置，文章中draft: true表示文章是一篇草稿，草稿默认无法部署，可以将其值修改为false，或在之后的部署时添加--buildDrafts以启用草稿

启动服务#

# 启动 hugo
hugo server
# 以指定主题启动
hugo server --theme=PaperMod
# 同时启用草稿
hugo server --theme=PaperMod --buildDrafts

启动成功后，通过http://localhost:1313/可以在本地预览效果

部署到 GitHub#

部署到 Gitee 也可以，但需要实名认证，方法略有不同，这里就不介绍了

首先需要在 GitHub 上新建一个空仓库，仓库名为UserName.github.io，其中UserName为自己GitHub的用户名

仓库必须选择公开（Public），否则部署上去也访问不到

其他选项默认即可，不用添加README.md

仓库建好之后，在站点目录下，记得替换UserName

hugo --theme=PaperMod --baseUrl="https://UserName.github.io" --buildDrafts

注：第一次执行，--theme=PaperMod参数一定要加，不然等会访问可能会加载不出 css 文件

然后进入生成的public目录下，将内容推送到 GitHub

cd public
git init
git add .
git commit -m "提交信息"
# UserName 为自己 GitHub 的用户名，替换即可
git remote add origin https://github.com:UserName/UserName.github.io.git
# 第一次 push 必须加上 -u master，以后再提交就只执行 git push 即可
git push origin -u master

推送完成后，访问https://UserName.github.io就可以看见博客内容了（GitHub 可能加载比较慢，需要多等一会儿）

添加新文章#

之后再添加新文章还是同样的方法，先新建文章

hugo new post/second-post.md

然后修改内容，修改完成后

hugo --theme=PaperMod --baseUrl="https://UserName.github.io" --buildDrafts

记得替换UserName，如果不想显示草稿，不加--buildDrafts即可
